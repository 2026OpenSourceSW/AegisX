# Agent Instructions

This repository follows `CONTRIBUTING.md` as the source of truth for branch, PR, review, and verification rules.

When working in this repository:

- Start from `develop` unless the user explicitly asks for a release or promotion branch.
- Use focused branches named `docs/*`, `feature/*`, `fix/*`, or `chore/*`.
- Do not push directly to `main` or `develop`.
- Open one pull request per coherent implementation, documentation, or workflow unit.
- Record verification evidence in the PR body before merge.
- Preserve upstream PentAGI legal attribution and third-party notices.
- Treat Shannon as an external integration unless maintainers explicitly approve a different license boundary.
- Keep README changes last when active implementation or verification work is still moving.
